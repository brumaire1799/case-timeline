# 案件时间轴生成器

将案件材料转化为清晰的时间轴，输出 HTML 网页（投屏演示）和 PPT（可编辑提交）。

---

## 输入模式

### 模式 A：结构化时间线文本

律师直接给出已整理好的结构化时间线，每行包含：
- **date**：时间（`YYYY年MM月DD日` 格式）
- **party**：归属方
- **event**：事件描述

直接跳到「第二步：确认当事人组别和上下方」。

### 模式 B：原始案件材料

律师提供原始文档（PDF、Word、Excel、JPG/PNG 图片、聊天记录截图等），或口述案情。需要 AI 从中提取时间节点。

**支持的格式**（全部通过 Claude Code 内置 Read 工具处理，无需额外安装组件）：
- PDF → 用 Read 工具读取（含扫描件 OCR 视觉理解）
- Word (.docx) → 用 Read 工具读取
- Excel (.xlsx) → 用 Read 工具读取
- 图片 (.jpg/.png) → 用 Read 工具读取（OCR 提取图中文字）
- 口述 → 逐条记录关键事件和时间，追问具体日期

**提取后必须经律师确认**，律师可手动增删改事件、调整时间和归属方。

---

## 工作流程

### 第一步：接收输入

- 模式 A：直接进入第二步
- 模式 B：读取材料，提取时间节点（提取指南见 [references/timeline-extraction.md](references/timeline-extraction.md)）

### 第二步：确认当事人组别和上下方

> **支持两方或多方当事人。** 案件可能涉及原告一/原告二、被告一/被告二/被上诉人三等多方当事人，每方独立配置颜色和位置。

**2.1 列出所有当事人**

从已提取的事件中统计所有唯一的 `party` 值，以清单形式展示给用户：

```
检测到以下当事人：
  1. 原告一
  2. 原告二
  3. 被告一
  4. 被告二
  5. 法院
```

询问用户：是否有遗漏、需要合并、或需要拆分的当事人？

**2.2 默认配色方案**

为每方自动分配默认颜色和位置：

- **默认位置**：名称含"原告/上诉人/申请人"的默认在**上方**，含"被告/被上诉人/被申请人"的默认在**下方**，法院/第三方默认在下方
- **默认颜色**：上方方从暖色系（红/橙/淡紫/金）依次分配，下方方从冷色/暗色系（黑/蓝/绿/紫）依次分配，法院/第三方用灰色

```
| 当事人   | 颜色     | 位置   |
|----------|---------|--------|
| 原告一   | #CC0000 正红  | 上     |
| 原告二   | #FF8C00 橙色  | 上     |
| 原告三   | #B39DDB 淡紫色 | 上     |
| 被告一   | #333333 炭黑  | 下     |
| 被告二   | #1A5276 深蓝  | 下     |
| 被告三   | #2E7D32 绿色  | 下     |
| 法院     | #666666 中灰  | 下     |
```

> **注意**：法院行为、第三方事件等，归入启动该程序或主导该行为的一方。如无法明确归属，归入"法院"或"第三方"。

**2.3 用户自定义配色**

默认方案展示后，逐一询问用户是否调整每方的颜色和位置：

- 逐方询问：**"原告一 使用 #CC0000 正红，可以吗？"** 如用户不满意，提供以下选项：
  - 从调色板中指定颜色（参考 [references/party-color-palette.md](references/party-color-palette.md)）
  - 直接输入任意十六进制色值（如 `#2E86C1`）
  - 描述期望颜色（如"深蓝""浅紫"），AI 映射为相近色值
- 逐方询问：**"原告一 放在上方，可以吗？"** 如需调整，直接指定上下位置
- 所有当事人确认完毕后，汇总展示最终配置表

**2.4 两方场景（简化为原流程）**

如果只有两方当事人，按以下默认规则快速确认：
- 询问用户选择组别类型：**原告 vs 被告** / **上诉人 vs 被上诉人** / **申请人 vs 被申请人**
- 默认：上诉方在上（红色 #CC0000），被上诉方在下（黑色 #333333）
- 确认时同样支持逐方自定义颜色和位置

### 第三步：确认时间轴（模式 B 必须；模式 A 可选）

将提取的时间轴以表格展示：

```
| 序号 | 时间           | 事件                           | 归属方 |
|------|---------------|-------------------------------|--------|
| 1    | 2025年1月10日  | 原告一与被告一签订《采购合同》    | 原告一  |
| 2    | 2025年2月15日  | 被告二将设备装车发货            | 被告二  |
| ...  | ...           | ...                            | ...    |
```

询问用户：
- 是否有遗漏的关键事件？
- 时间是否准确？
- 事件描述是否需要调整？
- 归属方标注是否正确？

**等待用户确认后再进入下一步。**

### 第四步：选择输出格式

- **HTML 网页**（推荐）：横向时间轴网页，每页最多 18 个事件。深色背景白色幻灯片，适合投屏。支持键盘翻页（← →）、全屏（F）、一键打印 PDF（Ctrl+P）。
- **PPT**：圆角矩形卡片式布局，黑体 8pt 左对齐，所有事件在一页内横向排列（宽度自动扩展）。每个文本框可独立编辑。

默认同时生成两种格式。

### 第五步：生成

#### HTML 网页

```bash
node scripts/generate_timeline_html.js events.json 案件时间轴.html
```

纯 Node.js，无需依赖。

**输入格式**（JSON 文件）：

```json
{
  "title": "（2025）沪XXXX民初XXX号 案件大事记时间轴",
  "parties": [
    {"name": "原告一", "color": "#CC0000", "position": "top"},
    {"name": "原告二", "color": "#FF8C00", "position": "top"},
    {"name": "被告一", "color": "#333333", "position": "bottom"},
    {"name": "被告二", "color": "#1A5276", "position": "bottom"},
    {"name": "法院", "color": "#666666", "position": "bottom"}
  ],
  "events": [
    {"date": "2025年1月10日", "event": "原告一与被告一签订《采购合同》", "party": "原告一"},
    {"date": "2025年2月15日", "event": "被告二将设备装车发货", "party": "被告二"},
    {"date": "2025年3月1日", "event": "原告一、原告二共同委托律师发送《律师函》", "party": "原告一、原告二"}
  ]
}
```

> **向后兼容**：也支持旧的两方格式（`topParty`/`bottomParty`），脚本会自动转换。
> **联合事件**：`party` 字段支持用顿号或逗号分隔多方（如 `"原告一、原告二"`），HTML 使用 CSS 渐变渲染多色左边框，PPT 使用扇形切割圆点。

**HTML 布局**：
- 16:9 幻灯片式页面，深色背景衬托白色卡片
- Flexbox 列布局，列宽按文本长度 + 事件间距（sqrt 天数）权重分配
- 卡片 min-height 2.5cm，高度不足时自动增长
- 每方独立颜色：卡片左边框、箭头、圆点均使用该方颜色
- 联合事件（多方 party）使用 CSS 渐变（从第一方颜色过渡到最后一方颜色）
- 连接线 1.5pt + 三角箭头，从文本框指向圆点（2.2cm）
- 时间轴主线 4.5pt 黑色 + 右端箭头
- 圆点绝对定位在轴线上，日期在圆点下方
- 键盘 ← → 翻页，F 键全屏，Ctrl+P 打印 PDF

#### PPT

```bash
~/miniconda3/bin/python3 scripts/generate_timeline_pptx.py events.json 案件时间轴.pptx
```

依赖 `python-pptx`（首次使用需 `pip install python-pptx`）。

输入格式同 HTML（同一个 JSON 文件）。

**PPT 布局**：
- 16:9 宽屏，高度固定 5.625"，宽度按事件数自动扩展（最大 56"）
- 所有事件横向排列在同一页，宽度不足时自动缩放
- 列布局，列宽按文本长度 + 事件间距（sqrt 天数）权重分配
- 卡片最小高度 2.5cm，不足时自动增长
- 每方独立颜色：卡片轮廓线、连接箭头、圆点均使用该方颜色
- 卡片白色填充 + 1.5pt 彩色轮廓线
- 事件文字：黑体 8pt 加粗左对齐，1 倍行距
- 连接线 1.5pt + 三角箭头，从文本框指向圆点
- 时间轴主线 4.5pt 黑色 + 右端箭头
- 图例按上下方分组显示所有当事人
- 每个文本框在 PowerPoint 中可独立编辑

### 第六步：交付

生成文件后提醒用户：
- **HTML**：浏览器打开，键盘翻页、全屏投屏（F 键）、一键打印 PDF，适合演示和提交法院
- **PPT**：每个文本框均可独立编辑，黑体 8pt，适合打印提交和进一步修改

## 时间格式规范

- 标准格式：`YYYY年MM月DD日`
- 不精确日期：如只有年份写 `2025年`，只有年月写 `2025年3月`
- 时间段：如"2025年1月至3月"，拆为起止两个节点
- 事件描述：控制在一两句话内，客观中性，不添加主观法律评价

---

## 归属方判断规则（模式 B 提取用）

- 单方行为（付款、交付、通知、催告等）：按行为主体判断
- 合同签订/协议签署：归入发起方或主要权利方
- 法院立案/开庭/判决：归入启动程序的一方（如立案归原告方，上诉归上诉方）
- 沟通/协商：归入发起沟通的一方
- 第三方事件（鉴定、评估等）：归入申请/启动该程序的一方
- **多方案件**：同一方当事人可能有多个（如原告一、原告二），需逐一区分标注
- **联合事件**：如果某一事件由多方共同参与（如原告一和原告二共同发函），`party` 字段用顿号或逗号分隔各方名称，如 `"原告一、原告二"`。生成时：
  - **HTML**：卡片左边框、连接箭头、圆点自动使用多色渐变（从第一方颜色过渡到最后一方颜色）
  - **PPT**：圆点使用扇形切割，每个扇区分配一方颜色

## 技术说明

### HTML 网页生成
脚本：`scripts/generate_timeline_html.js`，纯 Node.js，无需外部依赖。
- 支持多方当事人（`parties` 数组），每方独立颜色和上下位置
- 向后兼容旧的 `topParty`/`bottomParty` 两方格式
- 支持联合事件（多方 party 用 `、` 分隔），CSS 渐变多色渲染
- CSS Flexbox 精确布局，列宽按文本长度 + sqrt 间距权重，不重叠
- 每页最多 18 个事件，自动平衡分页
- 深色背景 + 白色幻灯片，适合投屏
- 键盘翻页、全屏、一键打印 PDF

### PPT 生成
脚本：`scripts/generate_timeline_pptx.py`，使用 `python-pptx` 库。
- 支持多方当事人（`parties` 数组），每方独立颜色和上下位置
- 向后兼容旧的 `topParty`/`bottomParty` 两方格式
- 圆角矩形卡片 + 真箭头连接线
- 所有事件在一页内横向排列，宽度自动扩展
- 每个文本框独立可编辑
- 支持联合事件（多方共用圆点扇形切割）
- 颜色从 party 配置自动映射
