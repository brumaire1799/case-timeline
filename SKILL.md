# 案件时间轴生成器

将案件材料转化为清晰的时间轴，输出 HTML 网页（投屏演示）和 PPT（可编辑提交）。

---

## 输入模式

### 模式 A：结构化时间线文本

律师直接给出已整理好的结构化时间线，每行包含：
- **date**：时间（`YYYY年MM月DD日` 格式）
- **party**：归属方
- **event**：事件描述

直接跳到「第二步：确认当事人组别和上下方」。

### 模式 B：原始案件材料

律师提供原始文档（PDF、Word、Excel、JPG/PNG 图片、聊天记录截图等），或口述案情。需要 AI 从中提取时间节点。

**支持的格式**（全部通过 Claude Code 内置 Read 工具处理，无需额外安装组件）：
- PDF → 用 Read 工具读取（含扫描件 OCR 视觉理解）
- Word (.docx) → 用 Read 工具读取
- Excel (.xlsx) → 用 Read 工具读取
- 图片 (.jpg/.png) → 用 Read 工具读取（OCR 提取图中文字）
- 口述 → 逐条记录关键事件和时间，追问具体日期

**提取后必须经律师确认**，律师可手动增删改事件、调整时间和归属方。

---

## 工作流程

### 第一步：接收输入

- 模式 A：直接进入第二步
- 模式 B：读取材料，提取时间节点（提取指南见 [references/timeline-extraction.md](references/timeline-extraction.md)）

### 第二步：确认当事人组别和上下方

询问用户选择当事人组别（三选一）：
- **原告 vs 被告**
- **上诉人 vs 被上诉人**
- **申请人 vs 被申请人**

然后确认：
- **哪一方在时间轴上方（红色字体）？**
- **哪一方在时间轴下方（黑色字体）？**

> 法院行为、第三方事件等，归入启动该程序或主导该行为的一方。

### 第三步：确认时间轴（模式 B 必须；模式 A 可选）

将提取的时间轴以表格展示：

```
| 序号 | 时间           | 事件                           | 归属方 |
|------|---------------|-------------------------------|--------|
| 1    | 2025年1月10日  | 原告与被告签订《采购合同》       | 原告    |
| 2    | 2025年2月15日  | 被告将设备装车发货              | 被告    |
| ...  | ...           | ...                            | ...    |
```

询问用户：
- 是否有遗漏的关键事件？
- 时间是否准确？
- 事件描述是否需要调整？
- 归属方标注是否正确？

**等待用户确认后再进入下一步。**

### 第四步：选择输出格式

- **HTML 网页**（推荐）：横向时间轴网页，每页最多 18 个事件。深色背景白色幻灯片，适合投屏。支持键盘翻页（← →）、全屏（F）、一键打印 PDF（Ctrl+P）。
- **PPT**：圆角矩形卡片式布局，黑体 8pt 左对齐，每页最多 16 个事件。每个文本框可独立编辑。

默认同时生成两种格式。

### 第五步：生成

#### HTML 网页

```bash
node scripts/generate_timeline_html.js events.json 案件时间轴.html
```

纯 Node.js，无需依赖。

**输入格式**（JSON 文件）：

```json
{
  "title": "（2025）沪XXXX民初XXX号 案件大事记时间轴",
  "topParty": "原告",
  "bottomParty": "被告",
  "events": [
    {"date": "2025年1月10日", "event": "原告与被告签订《采购合同》", "party": "原告"},
    {"date": "2025年2月15日", "event": "被告将设备装车发货", "party": "被告"}
  ]
}
```

**HTML 布局**：
- 16:9 幻灯片式页面，深色背景衬托白色卡片
- 每页最多 18 个事件，自动均匀分页
- 上方方 → 红色，下方方 → 黑色
- 文本框固定 min-height 2.5cm，宽度按字数自适应
- 连接线 1.5pt + 三角箭头，从文本框指向圆点（2.2cm）
- 时间轴主线 4.5pt 黑色 + 右端箭头
- 圆点绝对定位在轴线上，日期在圆点下方
- 键盘 ← → 翻页，F 键全屏，Ctrl+P 打印 PDF

#### PPT

```bash
~/miniconda3/bin/python3 scripts/generate_timeline_pptx.py events.json 案件时间轴.pptx
```

依赖 `python-pptx`（首次使用需 `pip install python-pptx`）。

输入格式同 HTML（同一个 JSON 文件）。

**PPT 布局**：
- 16:9 宽屏（10" × 5.625"）
- 每页最多 16 个事件，自动均分
- 上方方 → 红色，下方方 → 黑色
- 圆角矩形卡片：填充底色 + 1.5pt 彩色轮廓，固定高度 2.5cm
- 事件文字：黑体 8pt 加粗左对齐，1 倍行距
- 连接线 1.5pt + 三角箭头，从文本框指向圆点
- 时间轴主线 4.5pt 黑色 + 右端箭头
- 每个文本框在 PowerPoint 中可独立编辑

### 第六步：交付

生成文件后提醒用户：
- **HTML**：浏览器打开，键盘翻页、全屏投屏（F 键）、一键打印 PDF，适合演示和提交法院
- **PPT**：每个文本框均可独立编辑，黑体 8pt，适合打印提交和进一步修改

## 时间格式规范

- 标准格式：`YYYY年MM月DD日`
- 不精确日期：如只有年份写 `2025年`，只有年月写 `2025年3月`
- 时间段：如"2025年1月至3月"，拆为起止两个节点
- 事件描述：控制在一两句话内，客观中性，不添加主观法律评价

---

## 归属方判断规则（模式 B 提取用）

- 单方行为（付款、交付、通知、催告等）：按行为主体判断
- 合同签订/协议签署：归入发起方或主要权利方
- 法院立案/开庭/判决：归入启动程序的一方（如立案归原告方，上诉归上诉方）
- 沟通/协商：归入发起沟通的一方
- 第三方事件（鉴定、评估等）：归入申请/启动该程序的一方

## 技术说明

### HTML 网页生成
脚本：`scripts/generate_timeline_html.js`，纯 Node.js，无需外部依赖。
- CSS Flexbox 精确布局，不重叠
- 每页最多 18 个事件，紧凑卡片式设计
- 深色背景 + 白色幻灯片，适合投屏
- 键盘翻页、全屏、一键打印 PDF

### PPT 生成
脚本：`scripts/generate_timeline_pptx.py`，使用 `python-pptx` 库。
- 圆角矩形卡片 + 真箭头连接线
- 每页最多 16 个事件，自动平衡分页
- 每个文本框独立可编辑
- 上方/红、下方/黑
